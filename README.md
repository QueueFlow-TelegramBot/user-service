# User Service - SmartCampus Bot

Backend service for managing user profiles and authentication in the SmartCampus Bot system.

## Features

- User creation and management
- JWT token-based authentication
- PostgreSQL database integration
- Structured logging
- OpenAPI/Swagger documentation
- Docker support for local development

## Tech Stack

- **FastAPI** - Modern Python web framework
- **SQLAlchemy** - ORM for database operations
- **PostgreSQL** - Relational database
- **JWT** - Token-based authentication
- **Docker** - Containerization

## Prerequisites

- Python 3.11+
- Docker and Docker Compose (for containerized setup)
- PostgreSQL (if running without Docker)

## Quick Start with Docker

1. **Clone the repository**
   ```bash
   cd user-service
   ```

2. **Start services with Docker Compose**
   ```bash
   docker-compose up --build
   ```

3. **Access the API**
   - API: http://localhost:8000
   - Interactive docs: http://localhost:8000/docs
   - Health check: http://localhost:8000/health

## Local Development (without Docker)

1. **Create virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

2. **Install dependencies**
   ```bash
   pip install -r requirements/base.txt
   ```

   To update pinned dependencies after editing `requirements/base.in` or `requirements/dev.in`:
   ```bash
   make compile-requirements
   ```
   (`make compile-requirements` runs from inside the `requirements/` directory automatically.)

3. **Configure environment**
   ```bash
   cp .env.example .env
   # Edit .env with your database credentials
   ```

4. **Run the service**
   ```bash
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

## API Endpoints

### Public Endpoints
- `GET /` - Root endpoint with API info
- `GET /health` - Health check
- `POST /user` - Create new user
- `POST /user/token` - Generate authentication token
- `GET /user/{telegram_id}` - Get user by Telegram ID

### Protected Endpoints (require Bearer token)
- `PUT /user` - Update current user's display name
- `GET /user/me` - Get current user information

## Authentication

1. **Create a user**
   ```bash
   curl -X POST http://localhost:8000/user \
     -H "Content-Type: application/json" \
     -d '{"telegram_id": "123456789", "display_name": "John Doe"}'
   ```

2. **Get access token**
   ```bash
   curl -X POST "http://localhost:8000/user/token?telegram_id=123456789"
   ```

3. **Use token in requests**
   ```bash
   curl -X GET http://localhost:8000/user/me \
     -H "Authorization: Bearer YOUR_TOKEN_HERE"
   ```

## Environment Variables

See `.env.example` for all available configuration options:

- `DATABASE_URL` - PostgreSQL connection string
- `JWT_SECRET_KEY` - Secret key for JWT tokens
- `LOG_LEVEL` - Logging level (DEBUG, INFO, WARNING, ERROR)
- `ENV` - Environment (development, production)

## Database Schema

### Users Table
- `id` - UUID primary key
- `telegram_id` - Unique Telegram user ID
- `display_name` - User's display name
- `created_at` - Timestamp of creation
- `updated_at` - Timestamp of last update

## Logging

Structured JSON logging is enabled by default. Logs include:
- Timestamp
- Log level
- Service name
- Message

Example log output:
```json
{"time": "2026-03-13 10:30:00", "level": "INFO", "name": "user-service", "message": "User created: id=abc-123, telegram_id=123456789"}
```
