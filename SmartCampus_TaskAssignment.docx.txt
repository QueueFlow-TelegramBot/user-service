Functional Flows


Bot Subscribe Flow: (On startup) Telegram Bot -|Subscribe to notification.*|-> RabbitMQ


Create User Flow: (First time) User -|Join|-> Telegram Bot -|POST /user|-> User Service


Update User Flow: User -|Send Change Display Name|-> Telegram Bot -PUT /user-> User Service


Create Queue Room Flow: Queue Creator -|Send “Create Room [room_name]”|-> Telegram Bot -|POST /room|-> Scheduling Service -|return room_id|-> Telegram Bot -|show room_id|-> Queue Creator


Join Queue Room Flow: Queue Joiner -|Send “Join Room [room_id]”|-> 
* Telegram Bot -|POST /room/{room_id}/user|-> Scheduling Service -|return no_of_people_in_front, room_name, creator_name|-> Telegram Bot -|show no_of_people_in_front, room_name, creator_name|-> Queue Joiner
* Telegram Bot -|Publish to room.[room_id] the user_id|-> RabbitMQ


Request Next in Queue Flow: Queue Creator -|Send “Get Rooms”|-> Telegram Bot -|GET /room?creatorId={userId}|-> Scheduling Service -|return active_rooms|-> Telegram Bot -|show active_rooms|->  Queue creator -|Select room and “Get Next”|-> Telegram Bot -|POST /room/{room_id}/next|-> 
1. Scheduling Service -|Consume room.[room_id] the user_id|-> RabbitMQ
2. Scheduling Service -|Produce notification.[user_id] the room_id|-> RabbitMQ
3. RabbitMQ -|Invoke notification.[user_id]|-> Telegram Bot -|GET /room|-> Scheduling Service -|return room_name|-> Telegram Bot  -|Notify User with ID user_id|-> Queue Joiner


Topics


* notification.* (notification.[user_id]) - (SHORT LIVED MESSAGES, should be pushed to worker) - produced by the Scheduling Service upon a Queue Creator requesting the next Queue Joiner in a room, body is the ID of the room room_id, subscribed to by the Telegram Bot which will call a send message (notify) to the user with the ID being user_id (from the topic) Queue Joiner
* room.* (room.[room_id]) - (LONG LIVED MESSAGES, should be pulled by worker on request) - produced by the Telegram Bot upon a Queue Joiner joining the room with the ID room_id, body is the ID of the user joining user_id, polled by the Scheduling Service when a Queue Creator requests the next Queue Joiner in the list.
